Best Practices for Using Classes and ID's

Background

Before we delve into the best practices, let's first begin by discussing what a website is made of and the role ID's and classes play on a website.

Websites are made up of Hyper Text Markup Language (HTML) and Cascading Style Sheets (CSS).

HTML elements tell your web browser what information the website will contain. For example, a <p> element defines a paragraph of text and an <img> element defines an image. With this information the browser understands of how to treat each element and improve your experience of the website.

CSS adds style to your website. It tells your web browser how to present the information to you. For example, what font to use, what colour the background is and how large images are.

Classes and ID's

ID's and classes act as selectors on a website. Allowing the selection of specific elements and assigning attributes to them. This provides control over where styles are applied and how they are applied. Before you assign an ID or class to an element, it is important to understand the difference between the two selectors.

The ID selector allows you to define style rules that apply to a single element on the web page.

The class selector allows you to define style rules that apply to any element with a class attribute equal to a certain value.

Multiple ID selectors

The browser will apply multiple ID selectors if it has been supplied but it is best practice to use it with a single element. ID selectors have a higher hierarchy than classes and if they are both used on the same element, the ID will have priority. Below is an example of an ID with an attibute of border-radius="50%" being added to the first image as well as a class with an attribute of border-radius="0%". Notice how the image is round. This is an example of why an ID should be used on a single element only and used with caution. Click here to see an example in Codepen.

While ID's and classes appear to function in a similar way, ID's have greater functionality. ID's are intended to be unique and as a result have added functionality. They can be used as anchor points on a page which can be hyperlinked to. A user can click on a element that has an ID as a target, which will direct the user to a specific location on the web page.

In addition, JavaScript can access selectors and because ID's are designed for single use, expects to only find one element containing an ID it has been supplied. If the same ID has been added to multiple elements on a single web page, JavaScript will not behave as programmed and may cause bugs in your code.

Conclusion

When designing a website, think about common trends in the design. If you notice a pattern in the elements being used and the style you're giving them, then it's a good idea to use classes. If you are creating something unique, then using an ID is a good choice.
