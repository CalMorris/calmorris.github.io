Problem Solving Techniques with JavaScript

Building applications that fit a purpose and enhance people’s lives is extremely rewarding. It comes with a sense of pride and achievement but it also comes with overcoming obstacles. Computers require very direct and accurate information in the programming process. Mistakes in code are often not interpreted and this leads to bugs or unintended outcomes in the project.

Rather than muddling through the de-bugging process and hoping to accidentally find a solution, there are a range of techniques and tools which can be used to aid the problem solving process.

Pseudocode

Pseudocode is the first step. Before writing any code, taking time to understand the problem you are trying to solve and then writing down the steps needed to reach the outcome. This approach is written in common everyday language without technical syntax in the process. Removing syntax allows you to think about the process and prevent getting bogged down in what the syntax means and how it works.

There are many benefits to using pseudocode such as;

  • Better readability. Easy to understand steps for reaching the desired outcome. This can be shared with colleagues who can assist in the project without the need to ask questions as the work has been documented. Non-technical staff can review code and understand how it works without the need to learn a programming language.

  • Acts as a starting point. By designing the steps prior to writing any code, you have a clear path and starting point.

  • Bug detection. Since pseudocode is written in a human-readable format, it is easier to edit and discover bugs before actually writing a single line of code.

Guidelines to help write effective pseudocode:

  • Always capitalize the initial word

  • Have only one statement per line.

  • Indent to show hierarchy, improve readability, and show nested constructs.

  • Use the naming domain of the problem, not that of the implementation. E.g., “Append the last name to the first name” instead of “name = first+ last.”

  • Always end multiline sections using any of the END keywords (ENDIF, ENDWHILE, etc.).

  • Keep it simple, concise, and readable.

Console.log

Console.log is a function used to print results of an argument to the console. The console is a is a tool that logs information such as network requests, JavaScript, CSS, HTML and security error’s to name a few. Using the console.log() function in JavaScript, allows a developer to print the results of variables and functions. This is very useful in the build and debugging process. The ability to print results provides the opportunity investigate whether a task is performing as expected and is often one of the first steps in de-bugging as it can isolate part of the process.

Google

Google is a developers best friend. Programming is a vast field and it is impossible to accurately memorize all of the concepts, functions and use cases. Search engines like Google provide access to resources such as documentation which is regular maintained and community forums such as Stack Overflow where developers can post questions and seek help from other developers.

Rubber Ducky Method

Rubber ducky debugging. Yes you read that right. Rubber ducks have been helping programmers solve problems for decades. Before you pass judgement, it is worth understanding what the rubber ducky method is and why it works for developers around the world.

We have all been there. The frustration that we feel when code is not working or the output received is far from what you want. This is normal and you might try a range of solutions from; reading through your code line by line for bugs, perform tests with console.log and research for solutions on Google. What happens when all of these attempts are fruitless? What can you do?

Tell the duck your problem. Pretend the rubber ducky has no coding experience and explain the problem that your code is supposed to solve. Go into detail, and explain your code line by line. At some point, you will tell the duck what you are doing next and then realise why your code is not working.

The Psychology Behind Rubber Duck Method

To write the perfect code, you need to be as specific as you can because computers don’t understand vague instructions. The majority of the bugs a programmer faces in their code are caused due to the lack of providing explicit information and instruction to the software.

While explaining a concept or a problem to someone (or something), there are two subtle but ground-breaking shifts in your brain.

The first shift slows down your thinking. You’re able to describe the details more precisely without missing out on any information.

The second shift blends into the first one by the assumption that you’re talking to a rubber duck that understands nothing about the concept of coding bug.

By assuming that the duck doesn’t know details of the problems as clearly as you do, you’re forced to churn out the nitty-gritty bits and may find a solution in the same shift. Often it is a silly mistake in the code that was owed to speed and negligence.

Personal application of problem solving techniques

While practicing manipulating objects in the JavaScript Kata exercises at Dev Academy, one task included accessing an object which contained a list of students and their grades. The task included adding a score to a student. To perform this, the function needed to access the object, the individual by name, the testScores property and then the array of results.

Below is an example of the object:

Let gradebook = {
  Joseph: { testScores: [ 80, 70, 70, 100 ] },
  Susan: { testScores: [ 85, 80, 90, 90 ] },
  Wiremu: { testScores: [ 75, 70, 80, 75 ] },
  Elizabeth: { testScores: [ 100, 90, 95, 85 ] }
}

To access properties in an object, the dot(“.”) and square bracket (“[ ]”) notation can be applied. While attempting to access the scores, I used the dot natation and console.log'd it like this:

function(gradebook, name, score) {
  console.log(gradebook.name.testScores);
}

This returned an error:

TypeError: Cannot read property 'testScores' of undefined.

In response I copied and pasted the error message into google and found no obvious answer’s. After that approach did not work I began googling dot and bracket notation for JavaScript and read articles on how it is meant to be used. I found that dot notation cannot be used to access an object with a variable because it is not a property. So instead I used bracket notation on the variable and then used console.log to test this.

function(gradebook, name, score) {
  console.log(gradebook[name].testScores);
}

This worked perfectly and I was able to complete the challenge.
