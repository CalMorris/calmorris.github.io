Best practices when using using classes and ids

This post is intended for beginners with no prior knowledge or experience

Before we delve into the best practices, lets first begin by discussing what a website is made of and the role id’s and classes play on a website.

Websites are made up of Hyper Text Markup Language (HTML) and Cascading Style Sheets (CSS).

HTML elements that tell your browser what information the website will contain. For example, a <p> element defines a paragraph of text and an <img> element defines an image. With this information the browser has an understanding of how to treat each element and improve your experience of the website.

CSS adds style to your website. It tells your web browser how to present the information to you. For example, what font to use, what colour the background is and how large images are.

The HTML elements provide the structure to your website but their default styling is quite bland. Here is an example of a website with no styling:

As you can see it is not very appealing and there is a lot of room for improvement. Now the we have some structure, we can begin adding our styling our site.

CSS Styles can be applied to elements, but the drawback with this approach is that we are limited in what we can select. For example, I have 3 images of dogs and I want the second image to be round and with a black border. When I style the <img> element to be round with a black border, I get this result:

This happens because all of the <img> elements were selected and the browser applied the styling to all of the images. This is a very common scenario when building a website and an example of where classes and id’s are very useful.

Id’s and classes act as selectors on a website. Allowing you to select specific elements and assign attributes to them. This provides control over where styles are applied and how they are applied. Before you assign an id or class to an element you must first understand the difference between the two selectors.

The id selector allows you to define style rules that apply to a single element on the web page.

The class selector allows you to define style rules that apply to any element with a class attribute equal to a certain value.

You can assign a class or id to an element like this <img class='class-name'> or <img id=’id-name>.

Below is an example of classes and an id being assigned to three images. The first and third images are assigned a class to give them a black border. The second image has been assigned with an id to make the image round along with a black border.

The browser will apply multiple id selectors if it has been supplied but it is best practice to use it with a single element. Id selectors have a higher hierarchy than classes and if they are both used on the same element, the id will have priority. Below is an example of an id being added to the first image as well as a class. Notice how the image is now round. This is why an id should be used on a single element only and used with caution.
